Project Title: Agentic AI – L1 Order Issue Resolution Copilot (POC)

Build a professional Streamlit-based demo application that simulates an Agentic AI-powered L1 Support Assistant for tracking and resolving order issues in an enterprise order management ecosystem.

The application should demonstrate how AI reduces manual effort for L1 agents by automatically diagnosing order delays, identifying integration failures, and recommending or executing corrective actions.

🎯 Objective

Simulate a real-world L1 support scenario where:

Customers raise order-related queries.

L1 agents must investigate across multiple backend systems.

Orders may be stuck, delayed, or failing due to integration issues.

The AI assistant should:

Accept an Order ID.

Retrieve order status from multiple mock systems.

Detect delays and SLA breaches.

Analyze integration failures.

Provide root cause reasoning.

Recommend next best resolution steps.

Optionally simulate auto-remediation actions.

Generate a customer-ready response summary.

🏗 Technology Stack

Python

Streamlit

Groq (LLM – llama3-70b or latest)

Autogen (for multi-agent reasoning simulation)

Mock order dataset (JSON/CSV)

Logging module for tracking L1 interactions

Plotly for dashboards

Ensure clean, modular architecture.

🧠 Agentic AI Structure (Simulated)

Implement multi-agent orchestration using Autogen:

1️⃣ Order Investigation Agent

Fetches order details from Commerce, ERP, Payment, and Logistics (mock data).

2️⃣ SLA Monitoring Agent

Checks processing time vs SLA thresholds.

Flags at-risk or breached orders.

3️⃣ Integration Diagnostic Agent

Reviews simulated system logs.

Detects payment failure, ERP rejection, logistics congestion, API timeout, etc.

4️⃣ Resolution Recommendation Agent

Suggests:

Retry ERP submission

Re-trigger payment validation

Escalate to L2

Raise ServiceNow ticket (mock)

Can simulate automated corrective action.

5️⃣ Customer Communication Agent

Generates a clear, professional explanation for the customer.

Avoids technical jargon.
